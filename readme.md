# ProfileAI

ProfileAI is a conversational AI tool that allows you to chat with any LinkedIn profile. It scrapes a user's public LinkedIn data, processes it into a structured knowledge base, and uses a Retrieval-Augmented Generation (RAG) model to answer your questions about their career, skills, experience, and more.

## Architecture

The application is composed of three main components:

1.  **Frontend (`index.html`):** A single-page, vanilla HTML, CSS, and JavaScript interface that provides the user with a chat-like experience. It communicates with the two backend services to fetch and query profile data.

2.  **Data Scraping Backend (`app.py`):** A Flask server responsible for:
    *   Receiving a LinkedIn username from the frontend.
    *   Using the Bright Data API to scrape the public profile information.
    *   Processing the raw JSON data into structured text chunks (about, education, work, certifications, activities) via `embedding.py`.
    *   Storing these chunks in the `userData/` directory.
    *   Triggering the creation of a vector database from these chunks.

3.  **RAG Model Backend (`rag_model.py`):** A separate Flask server that handles the AI logic:
    *   It loads the text chunks generated by the data backend.
    *   It uses Hugging Face sentence transformers (`all-MiniLM-L6-v2`) to create vector embeddings.
    *   These embeddings are stored in a persistent ChromaDB vector store.
    *   It exposes a query endpoint that uses a local LLM (via Ollama) to answer questions based on the retrieved context from the vector database.

## Features

*   **Deep Profile Analysis:** Extracts comprehensive details including user summary, work experience, education, certifications, and recent activity.
*   **Conversational AI:** Engage in a natural conversation to get insights about a profile using a RAG pipeline with a local `llama3` model.
*   **Dynamic Knowledge Base:** A unique vector database is created for each analyzed profile, ensuring contextually accurate responses.
*   **Simple Web Interface:** An intuitive UI to input a LinkedIn username and start chatting immediately.

## Tech Stack

*   **Backend:** Python, Flask
*   **AI/ML:** LangChain, Ollama, ChromaDB, Hugging Face Transformers
*   **Frontend:** HTML, CSS, JavaScript
*   **Data Scraping:** Bright Data API

## Setup and Installation

Follow these steps to get the project running locally.

### Prerequisites

*   Python 3.8+
*   [Ollama](https://ollama.com/) installed and running.
*   The `llama3` model pulled via Ollama:
    ```sh
    ollama pull llama3
    ```

### 1. Clone the Repository

```sh
git clone https://github.com/0xZ15H4N/Digital-Persona.git
cd Digital-Persona
```

### 2. Install Dependencies

It's recommended to use a virtual environment.

```sh
python -m venv venv
source venv/bin/activate  # On Windows, use `venv\Scripts\activate`
```

Install the required Python packages:

```sh
pip install flask flask_cors requests python-dotenv langchain langchain_community langchain_core langchain_chroma sentence-transformers
```

### 3. Configure Environment Variables

Create a `.env` file by copying the sample file:

```sh
cp .env.sample .env
```

Open the `.env` file and add your Bright Data API key:

```
api_key=YOUR_BRIGHTDATA_API_KEY
```

### 4. Run the Backend Servers

You need to run the two Flask servers in separate terminal windows.

**Terminal 1: Data Scraping Backend**

```sh
flask --app app run --port 5000
```

**Terminal 2: RAG Model Backend**

```sh
flask --app rag_model run --port 5001
```

### 5. Launch the Frontend

Open the `index.html` file in your web browser.

```sh
python -m http.server 8000
```


## Usage

1.  Once the application is open in your browser, enter a valid LinkedIn profile username (e.g., `john-doe-123`) into the input field.
2.  Click the "Analyze" button.
3.  The application will fetch the profile data, process it, and create a vector database. The status will be displayed on the screen.
4.  Once complete, you will be taken to the chat view.
5.  You can now ask questions about the profile, such as "What are their top skills?" or "Summarize their work experience."
